{% comment %} The one detail overlay. It is a scrim and nothing else: the card htmx swaps in carries role/aria-labelledby/data-url-param, so closing never needs to know what is inside. Before this, three containers each hard-coded which query param their close function stripped, and the empty #panel div wore aria-labelledby="panel-title" permanently — pointing a screen reader at the slice title even when a ticket was open. .window on escape is required: without it Esc dies the moment focus leaves the card (clicking the page behind an undimmed slide-over did exactly that). The innerHTML guard is equally required, because .window also fires when nothing is open. {% endcomment %}
